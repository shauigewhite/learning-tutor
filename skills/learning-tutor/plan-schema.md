# 教导计划格式规范

## 与 Claude Code 记忆系统的融合

教导计划文件也遵循标准记忆文件格式，使用 frontmatter 确保可被检索系统发现。

## 有教材的场景（用户提供了 URL 或文档）

1. 抓取教材内容，**只提取目录结构和各章节摘要**，不保存全文
2. 根据目录生成教导计划，保存到 `memory/learn/{领域}-plan.md`
3. 记录当前进度（学到哪一章哪一节）
4. 每次学习时从断点继续，需要教某一节时再实时抓取该节内容
5. 教导计划可根据用户反馈随时调整（跳过或深入某些内容）

## 没有教材的场景（用户说"我想学 XXX"）

1. 先评估用户水平
2. 根据水平生成分阶段的教导大纲
3. 让用户确认大纲是否满意，可调整
4. 保存大纲到 `memory/learn/{领域}-plan.md`
5. 按大纲逐步教学，记录进度

## 计划文件格式

```markdown
---
name: plan-python
description: Python 教导计划：从基础语法到高级特性的学习大纲，当前进度在"装���器"章节
type: project
---

# Python 教导计划
来源：{URL 或"自主生成"}
上次学习：{日期}

## 进度
当前：{章节名}

## 大纲
- [x] 已完成的章节
- [ ] 当前章节 ← 当前
- [ ] 未开始的章节
```

**注意：**
- `type` 用 `project`（因为教导计划属于进行中的项目/学习活动��
- `description` 必须包含课程名和当前进度关键词
- 每次学习结束后更新 description 中的进度信息

## 精简原则

- 计划文件控制在 **30 行以内**（不含 frontmatter）
- 只保存目录级别的信息，不保存章节全文
- 每次学习结束后更新进度标记和"上次学习"日期
- 新建计划时同步在 MEMORY.md 中添加索引指针
