# 学习者档案格式规范

## 与 Claude Code 记忆系统的融合

学习档案存放在 memory 目录中，**必须遵循 Claude Code ���标准记忆文件格式**（含 frontmatter），这样：
- 档案会被记忆检索系统（Sonnet sideQuery）自动发现并在相关对话中加载
- 档案的 `description` 字段决��了检索相关性——要写得精准

### 关键规则
- 每个档案文件必须有 `name`、`description`、`type` 三个 frontmatter 字段
- `type` 统一用 `user`（因为学习档案属于用户个人信息）
- `description` 要包含领域关键词，确保用户提到该领域时能被检索到

## 两层档案结构

### 主档案：`memory/learner-profile.md`

记录用户基本信息和领域索引。控制在 **15 行以内**（不含 frontmatter）。

```markdown
---
name: learner-profile
description: 学习者主档案：用户背���、学习偏好、所有学习领域的索引和复习日期表
type: user
---

# 学习者档案
背景：[用户背景，如"后端开发者，3年经验"]
偏好：[学习偏好，如"喜欢类比，偏好代码示例"]

## 领域索引
| 领域 | 整体等级 | 上次学习 | 下次复习 |
|------|---------|---------|---------|
| Python | 3 | 2025-01-15 | 2025-02-01 |
| 机器学习 | 2 | 2025-01-20 | 2025-01-27 |
```

### 详细档案：`memory/learn/{领域}.md`

每个领域的知识点掌握明细。控制在 **30 行以内**（不含 frontmatter）。

```markdown
---
name: learn-python
description: Python 学习档案：知识点掌握明细（装饰器、生成器、列表推导式等），含等级、正确率和复习日期
type: user
---

# Python 学习档案

## 知识点掌握
| 知识点 | 等级 | 前测 | 后测 | 下次复习 |
|--------|-----|------|------|---------|
| 列表推导式 | 4 | 80% | 100% | 2025-02-15 |
| 装饰器 | 2 | 20% | 60% | 2025-01-25 |
| 生成器 | 1 | 0% | - | 待学习 |

## 需重点复习
装饰器（第2次循环后仍模糊）
```

**description 写法要求：** 必须包含该领域的核心关键词（如知识点名称），以便用户提到相关话题时被记忆检��系统命中。例如用户问"Python 装饰器怎么用"时，description 中的"装饰器"会触发该档案被加载。

## 掌握等级定义

| 等级 | 含义 | 间隔复习周期 |
|------|------|------------|
| 1 | 完全不了解 | - |
| 2 | 听过但说不清楚 | 3 天 |
| 3 | 能基本解释但不能灵活应用 | 7 天 |
| 4 | 能用自己的话解释并应用到新场景 | 14 天 |
| 5 | 能教别人，融会贯通 | 30 天 |

## 读取规��

1. 每次辅导开始时，先读取主档案了解用户特点和领域索引
2. 再根据当前学习主题读取对应的详细档案
3. 检查是否��知识点已到"下次复习"日期 — 如果有，优先安排复习
4. 跳过等级 4-5 的内容（除非到了复习日期），重点关注等级 1-3
5. **注意档案时效性**：如果档案"上次学习"距今超过 30 天，知识点掌握情况可能��过时，前测时应重新验证

## 更新��则

- 每次学习结束后更新对应的详细档案
- 根据后测结果更新每个知识点的等级和复习日期
- 如果是新领域，在主档案索引表中新增一行，并创建新的详细档案
- **同步更新 MEMORY.md 索引**：新增领域档案时，在 MEMORY.md 中添加一行指针（遵循记忆系统的标准格式）
- 用逗号分隔列表记录知识点，不写冗余备注
- **更新 description**：每次更新档案内容后，同步更新 frontmatter 的 description，确保包含最新的知识点关键词

## 间隔复习日期计算

`下次复习 = 今天 + 对应等级的复习周期`

等级提升时重置复习周期。等级下降时缩短周期（用新等级对应的周期）。
